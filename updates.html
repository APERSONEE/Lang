How the Lang update system works

What do updates usually contain?

Lang updates are automatically pushed to the server and you should not notice any down time. Updates can contain anything from bug fixes to total UI overhauls. The most common form of update, the routine update, typically contains bugfixes and maybe small UI tweaks, often requested by users. These updates can come from week to week, and potentially day to day depending on how busy the devs are :).

How are updates labeled?

Lang updates are labeled specifically to make it easier for users and the Lang team to see what an update contains. There are 3 numbers in the label of a lang update. For example, at the time of writing this, Lang is on version 3.1.1. The first number signifies the major version of Lang. We are currently on the 3rd major version of Lang (see below for previous versions). The differences between versions are huge and readily apparent. The second number is how many features were added from the start of the major version. Currently, only 1 feature has been added since the start of version 3 (Async requests to the server). The final number is how many small bugfix/UI tweaks were done since the start of a major version. It is also currently at 1.
